### 🔥 今日必看

- **Compaction as Epistemic Failure: How Agentic LLM Tools Fabricate Confirmed Results from Killed Processes**（arXiv 2607.13071，Hiroki Tamba）— 记录了 Claude Code 里一个具体故障模式：超时被杀掉的命令（exit code 143）残留的部分 stdout，在生成 compaction 摘要时被写成"已确认结果"，之后跨 session、跨模型版本被当作事实直接继承、不再复核；根源是把"终端里出现过"和"已经持久化"这两件事搞混了。跟 07-18 收录的 Governance Decay（compaction 让安全约束违规率从 0% 升到 30%）指向同一个风险面——context compaction 不只是有损，还可能主动伪造事实，对我们"compaction 只丢细节、不改变事实真伪"的默认假设构成直接挑战。

### 📄 新论文

#### KV Cache 压缩/驱逐
- **MemDecay: Region-Aware KV Cache Eviction for Efficient LLM Agent Inference**（arXiv 2607.10582，Venkatesha Matam, Keon Kim）— 指出 agent context 由系统指令、计划、用户轮次、检索文档、工具输出、中间推理等异质区域构成，而现有 eviction 策略对所有 token 套用同一套 attention/recency 规则、忽视了 orchestrator 早已知道的语义结构；MemDecay 按区域类型分配不同的免训练驱逐策略。

#### Systems / Serving
- **Benchmarking KV-Cache Optimizations across Task Quality and System Performance for Long-Context Serving**（arXiv 2607.05399，Nikita Agrawal, Ruben Mayer）— 面向长上下文 serving 场景，对 KV cache 量化/剪枝/合并等优化手段做任务质量与系统性能双轴的系统性 benchmark，PVLDB 论文并开源评测代码；作为横向对比参照资源记录，非单点新方法。

#### Agentic Context 管理
- **CoACT: Action-Preserving Observation Compression for Coding Agents**（arXiv 2607.02911，Haorui Chen 等，清华）— 针对 coding agent 里工具观测持续累积推高推理成本的问题，提出 next-action preservation（NAP）判据：只要压缩后的观测仍能诱导出和原始观测相同的下一步动作就算合格；用教师模型生成候选、按 NAP 过滤+长度奖励筛选来监督一个轻量压缩器，SWE-bench Verified 上三个 agent 模型平均总 token 消耗降 33.0%、任务效果接近未压缩基线。7 月初已挂出但此前几轮均未搜到，本轮补录。
