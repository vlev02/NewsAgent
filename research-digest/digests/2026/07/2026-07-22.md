### 🔥 今日必看

- **Plans Don't Persist: Why Context Management Is Load Bearing for LLM Agents**（arXiv 2606.22953，Snowflake AI Research）— 用 replay pairing 诊断法证明标准 agent 并不会把 plan 内化为持久状态：plan 写入后一步内隐藏态相似度飙升，随后一个 action-observation 步骤内骤降 4.1x（HotpotQA 上降 12.4x），朴素驱逐 plan 让 ALFWorld 成功率降 34.7pp 且探针门控重浮现也无法完全恢复。直接挑战"只要保护住 plan 就能安全做 context compaction"的假设，和已收录的 Governance Decay（07-18）、Compaction as Epistemic Failure（07-20）构成同一威胁模型下的三重印证，值得放进风险清单。
- **Kamera: Unified Position-Invariant Multimodal KV Cache for Training-Free Reuse**（arXiv 2606.23581）— 量化指出朴素 KV 复用会丢失跨块条件化信息：单跳召回不受影响但多跳准确率被腰斩，此前的 position-independent cache 方案（面向单上下文/单图像复用）都没处理这个失效模式。提出训练-free 的低秩 conditioning patch 随位置无关 chunk 一起存储 + 精确 RoPE 重旋转，在 MLA/GQA/MHA 上统一为一个算子，直接呼应 R1 关于"多块拼接是否 scramble 位置"的问题，是对 CacheBlend/CacheClip 一脉的一次量化补强。

### 📄 新论文

#### KV Cache 压缩/驱逐
- **Still: Amortized KV Cache Compaction in a Single Forward Pass**（arXiv 2606.07878）— 训练一个针对冻结基座模型的小型逐层 Perceiver，一次前向即可产出压缩后的 K/V，兼顾轻量部署、约束下保留上下文、轨迹内可复用三重要求；Qwen/Gemma 上 8x-200x 压缩比、8k-128k 上下文范围内均处于速度-质量前沿，RULER 上超过最强 baseline 8-22 分。
- **UltraQuant: 4-bit KV Caching for Context-Heavy Agents**（arXiv 2606.20474）— 针对 context-heavy agent 场景长前缀跨多轮复用、并发决定 GPU 利用率的特点做 4-bit KV 量化：非对称 K/V 处理、Walsh-Hadamard 旋转、去掉 QJL，AMD CDNA4 上实现 FP8 query + FP4 KV 路径，把任务质量、缓存驻留、serving 吞吐联合评估而非孤立看压缩比。

#### KV Cache 复用 / Position Re-indexing
- **LazyAttention: Efficient Retrieval-Augmented Generation with Deferred Positional Encoding**（arXiv 2606.04302，ICML 2026）— 在 attention kernel 内核化延迟位置编码，实现零拷贝、位置无关的 KV 复用，单份物理 KV 拷贝可服务任意位置的多个逻辑请求；偏斜文档分布下相比 Block-Attention 降低 TTFT 1.37x、提升吞吐 1.40x，和 MiniPIC/HYPIC/CanonicalMerge 同属 position-agnostic 复用路线的又一实现。

#### Systems / Serving
- **SAC: Disaggregated KV Cache System for Sparse Attention LLMs with CXL**（arXiv 2606.19746）— 指出传统基于 RDMA 的分离式内存池对稀疏注意力模型仍要粗粒度搬回整段前缀 KV；利用 CXL 的 cache-line 粒度 load/store 语义按需只取 decode 阶段真正需要的 top-k KV 条目，DeepSeek-V3.2 + SGLang 上相比 RDMA 基线吞吐提升 2.1x、TTFT 降低 9.7x。
- **HERALD: High-Throughput Block Diffusion LLM Serving via CPU-GPU Cooperative KV Cache Retrieval**（arXiv 2606.21633）— 面向块扩散 LLM 的 KV cache CPU offload 场景，利用同一去噪块内相关 KV 条目跨步骤保持一致的特性摊薄 top-k 选择开销并与去噪过程重叠，5%-10% KV 预算下近无损精度；架构非标准自回归 LLM，作为 offload/retrieval 机制背景参考收录。

### 🏗️ 工程/生态动态

- **LMCache**：GitHub releases 页面今日（7/22）发布 v0.5.2，加入 MiniMax M3 支持和与 vLLM hybrid memory manager 的 CacheBlend 兼容性（watchlist 项目首次看到官方 serving 框架侧的一等公民支持）；前一天 7/20 的 v0.5.2rc1 预发布中包含"strided fused-K re-RoPE optimization for vLLM packed KV cache format"，直接对应 tier_0 的 RoPE re-rotation cache 方向，是本轮少见的算法层面工程更新，值得后续追踪细节。
- **SGLang**：v0.5.15（7/10）引入 IndexShare MTP，跨 draft step 复用 indexer top-k，长上下文下 draft-step 开销最多降 1.9x；v0.5.14（6/26）为 KDA/GDN 等线性注意力模型的 prefix cache 引入 int8 checkpoint pool，大幅提升 prefix-cache 容量，两者均与 KV cache 管理直接相关但发布时间已超过 24-48h 窗口，作为背景记录。
- **vLLM**：v0.25.0（7/11）/v0.25.1（7/14）延续 DCP+FP8 KV cache（MLA decode）、Triton INT4 per-token-head KV cache 量化等既有方向的迭代，本轮未见新的 KV cache 管理算法公告。
