### 🔥 今日必看

- **Adaptive Filtering of the KV Cache: Diagnosing and Correcting Structural-Role Bias in LLM Inference**（arXiv 2607.13205）— 把 H2O 类"累积 attention mass 排名保留"的 eviction 方法放到 schema 密集输入（嵌套 JSON）上做诊断：非内容的 sink 角色（分隔符/空白）携带的能量比任何内容角色高一个数量级，结构性 KEY token 被过度保留、速率约是 answer-carrying VALUE token 的 1.8 倍，5% 预算下 exact-match 从 88% 直接崩到 0%。反事实实验证明压低 KEY token 权重是最佳可部署过滤器；免重训、角色条件化分配叠加在 SnapKV 的窗口分数上，sub-20% 预算下补回 63%-98% 的 H2O 差距。这是目前看到的对"attention mass = token 重要性"假设最锋利的一次证伪。
- **Jet-Long: Efficient Long-Context Extension with Dynamic Bifocal RoPE**（arXiv 2607.07740，NVIDIA）— 现有零样本长上下文扩展方法通常固定一个 rescaling 因子，激进则牺牲短文本保真、保守则长文本失效。Jet-Long 用本地窗口保持 RoPE 原生保真、远程窗口的 rescaling 因子随当前序列长度动态调整，靠 inclusion-exclusion attention merge + 在线 RoPE 校正旋转让双焦构造几乎零额外开销，融合进单个 CuTe kernel 后长上下文 prefill 达 FA2 吞吐的 1.39 倍。Qwen3-1.7B/4B/8B 上 128K 上下文 RULER 较最强 baseline 提升 +2.03~+4.79pp。直接命中"位置管理/RoPE 外推"这条核心关注点。
- **KVEraser: Learning to Steer KV Cache for Efficient Localized Context Erasing**（arXiv 2606.17034，ICML 2026 Workshop on Impact of Memorization on Trustworthy Foundation Models Oral）— 指出 KV cache 上的事后擦除很难：一个 span 处理完之后，它的影响已经传播进后续所有 token 的缓存态里，精确擦除的代价取决于后缀长度而非被删 span 长度。这正是 agentic 场景里"过期检索事实/错误工具观测/被撤回的用户偏好/prompt injection 事后才被发现"的典型问题。KVEraser 只把被删区间的 KV 替换成学习到的 steering states、其余缓存原样复用，受控 benchmark 上 1K-32K token 范围内做到接近精确重算的效果，长文档 QA 上在近似方法里表现最好，且比全量重算快 3-4 倍。和昨天收录的 Leyline（声明式 span 编辑 + RoPE 校正）是同一方向的两种解法，值得对照。

### 📄 新论文

#### KV Cache 压缩/驱逐
- Information-Aware KV Cache Compression for Long Reasoning / InfoKV（arXiv 2606.26875）— 指出现有基于 attention 分数的 KV 压缩只捕捉近距离上下文关联，忽略预测不确定性等信息论信号；提出 Forward Influence 指标，发现高预测不确定性的 token 对远距离未来上下文的影响明显更强。InfoKV 把 token 级预测不确定性与逐层表征演化结合成熵分数，和 attention 分数一起用于压缩决策，在 Llama-3.1/3.2 与 DeepSeek-R1 的长推理 benchmark 上全面优于现有 attention-based 压缩方法。
- STAR-KV: Low-Rank KV Cache Compression via Soft Thresholding for Adaptive Rank Control（arXiv 2606.08382）— 本周作为 ICML 2026 Spotlight 被重新报道的低秩 KV 压缩工作（UC San Diego VVIP Lab + Dnotitia）。用可微 soft-thresholding 在 head/block 粒度自适应选秩，按 K/V 投影的敏感度分别用不同低秩分解策略，配合低秩感知的混合精度量化；单独低秩压缩即可达 75%，叠加量化后总体压缩最高 20x，attention 计算加速 6.9x、生成吞吐提升 3.1x。

#### Systems / Serving
- CacheWise: Understanding Workloads and Optimizing KVCache Management for Efficiently Serving LLM Coding Agents（arXiv 2606.16824，University of Washington）— 研究发现 coding agent 的 serving 行为和 chat workload 不同：session 会反复复用大前缀、造成持续性 KVCache 压力，常规 serving 策略处理不好。CacheWise 结合 prefix-aware 调度与基于工具调用元数据轻量预测的 reuse-aware eviction，在 vLLM 中实现并在实采 trace 上评测，驱逐次数降 2-2.6x，agent session 完成时间最多提速 3.5x。是 tier_1 系统层与 tier_2 agentic 场景的交叉工作。

#### Agentic Context 管理
- Memory as a Controlled Process: Learned Adaptive Memory Management for LLM Agents / MemCon（arXiv 2607.13591）— 指出现有 agent 记忆系统（graph 结构记忆、反思式 insight store 等）都靠固定的手工启发式规则访问记忆，而最优记忆行为其实高度依赖上下文，这是 agentic learning 的核心瓶颈。MemCon 把记忆操作（何时/取什么/取多少、何时注入蒸馏计划、何时巩固或遗忘）建模成一个 MDP，学一个在线策略自适应决策。
- CompactionRL: Reinforcement Learning with Context Compaction for Long-Horizon Agents（arXiv 2607.05378）— 现有 context compaction 通常是推理时的启发式摘要，和任务训练本身脱节。CompactionRL 用 PPO 把摘要生成变成模型可学习的一部分：从 group-wise 优化转向按压缩片段学习的 PPO 形式，依赖 critic 估计 token 级 advantage，联合优化任务执行和摘要生成，用 token 级 loss 归一化和跨轨迹 GAE。
- Less Context, Better Agents: Efficient Context Engineering for Long-Horizon Tool-Using LLM Agents（arXiv 2606.10209）— 少见的真实企业 ERP agent 场景端到端实验（而非 QA/coding benchmark，成功标准是业务定义的"zero residual"）：固定近期性驱逐策略（只保留最近 5 次工具调用）配合可选的单次摘要，在效果和效率上都优于全量保留 context，说明旧 context 有时是主动有害而非只是冗余——对"保留更多 context 总是更安全"这个默认假设提出挑战。
