### 🔥 今日必看

- **MiniPIC: Flexible Position-Independent Caching in <100LOC**（arXiv 2606.13126）—
  watchlist 项目本体的最新论文。核心思路是在 KV cache 里存"未旋转"的 K 向量,
  attention 内部按每个请求的逻辑 position 现场施加 RoPE,用不到 100 行核心引擎改动
  在同一个 vLLM 实例里实现多种 position-independent caching 方法。这跟 R1（多块
  re-index 是否 scramble）直接相关——它是把 rotation 推迟到 attention 内部而不是
  缓存后再修正,值得对比 ReClaim 的 delta-based rotary correction 路线是否更省或更贵。
- **GSRQ: Gain-Shape Residual Quantization for Sub-1-bit KV Cache**（arXiv 2607.01065）—
  把 KV cache 量化推到 sub-1-bit 区间,用增益-形状残差量化。跟压缩类基线（SnapKV/
  ChunkKV 走 token 级驱逐)是正交路线（量化 vs 驱逐),但极限压缩比可能影响我们对
  "剩余 token 数下限"的假设,值得跟踪是否会被主流 serving 框架采纳。
- **IntentKV: Cross-Turn Intent-Aware KV Cache Pruning for Agent Inference**
  （arXiv 2606.09916）— watchlist 项目,面向 agent 场景的跨轮次意图感知 KV 剪枝,
  维护 session 级 QueryMemory 并用 memory-attention 规则给历史 token 打分。是少数
  真正把"agentic"和"KV cache 管理"焊死在一起的工作,而不是脱钩的 context 摘要方案,
  对 tier_2 的评估基线有参考价值。

### 📄 新论文

#### KV Cache 压缩/驱逐
- **GSRQ**（2607.01065）— 增益-形状残差量化,把 KV cache 推向 sub-1-bit。
- **CompressKV**（2606.24467）— 语义检索引导的 KV 压缩,按 attention head 功能差异化打分,
  避免关键 token 被误逐。
- **VaSE (Value-Aware Stochastic KV Cache Eviction)**（2606.03928）— 面向推理模型的解码期
  KV 驱逐,声称比 R-KV 在 Qwen3-4B/14B 上分别高 4.4%/4.9%。
- **Recency/Frequency Adaptive KV Caching**（2606.21238）— serving 场景下按新鲜度/频率
  自适应调整缓存策略。

#### KV Cache 复用 / Position Re-indexing
- **MiniPIC**（2606.13126）— 见"今日必看"。
- **MEPIC: Memory Efficient Position Independent Caching**（arXiv 2512.16822）—
  同属 position-independent caching 方向,通过 RoPE fusion 去除位置编码依赖,
  实现跨 position/request/batch 的 chunk KV 复用,和 MiniPIC 形成"重设计 vs 小改动"
  的两种路线对照。
- **CacheSlide**（USENIX FAST'26,未见独立 arXiv 预印本）— watchlist 项目,主打
  跨 position 感知的 KV cache 管理,论文声称对比 SOTA baseline 有 3-4x 延迟/吞吐提升。

#### Systems / Serving
- **PrefillShare: A Shared Prefill Module for KV Reuse in Multi-LLM Disaggregated Serving**
  （arXiv 2602.12029）— 多模型 disaggregated serving 场景下的共享 prefill 模块,
  用于减少重复 KV 计算。

#### Agentic Context 管理
- **Self-GC: Self-Governing Context for Long-Horizon LLM Agents**（arXiv 2607.00692）—
  见"今日必看",本周搜到的最新一批 agentic context 论文里日期最近的一篇。
- **ACE: Pluggable Adaptive Context Elasticizer across Agents**（arXiv 2606.31564）—
  可插拔的自适应 context 伸缩模块,声称能跨不同 agent 框架复用。
- **Beyond Compaction: Structured Context Eviction for Long-Horizon Agents**
  （arXiv 2606.11213）— 提出结构化 context 驱逐(而非纯粹摘要压缩),声称长序列
  agent session 精度无明显退化;文中提到 DeepSeek-V3.2/Claude Code/GLM-5 各自不同的
  驱逐策略对比,值得作为 agentic tier_2 的背景参照系摘录进笔记。

### 🏗️ 工程/生态动态

- **vLLM 0.25.0**（2026-07-11 发布）— 新增 Streaming Parser Engine(统一 tool-call/
  reasoning 解析)、FP8/FP4 量化增强等,未见本次直接触及 KV cache eviction/position
  管理的改动,记录仅为版本追踪。
- **SGLang v0.5.15**（2026-07-10 发布）— HiCache 对 SWA/Mamba 等混合架构模型默认启用,
  与 tier_3 的 state space model 长上下文方向有交集,值得关注 HiCache 的驱逐策略细节。
- **LMCache** — 无过去 48 小时内的新 release;此前 4 月发布的 multiprocess 架构、
  1 月 P2P CPU 内存共享转正式产线是较早的里程碑,未见本周更新。
