### 🔥 今日必看

- **Cache Merging as a Convergent Replicated State for Multi-Agent Latent Reasoning**（arXiv 2607.01308）— 指出 Agent Primitives 的 BagMerge（沿序列轴拼接多 agent KV cache + RoPE re-encoding）是非交换的，输入顺序会显著影响结果且难以预判。提出 Canonical Merge：按中间层 mean K-norm 做内容确定性排序，让合并后的 cache 在任意输入排列下都是 byte-identical（在 Qwen3-1.7B/4B 上验证到 bit 级），把"多 agent 缓存合并"形式化成一个 CvRDT（可交换、可结合、幂等）。直接命中"多块 re-index 是否 scramble"这条核心关注点，而且给出了一个可验证的确定性解法。
- **Beyond Compaction: Structured Context Eviction for Long-Horizon Agents**（arXiv 2606.11213，提出 Context Window Lifecycle / CWL）— 让 agent 把轨迹标注成有类型、有依赖关系的 episode，用确定性、不依赖 LLM 的策略按优先级驱逐（已经落盘到环境里的 action episode 优先清，用户轮次和正在推理的探索性 context 优先留）。相比基于摘要的压缩，避免了不可预测的有损性、破坏因果结构、阻塞式模型开销和压缩导致的幻觉四个已知问题；在单个 agent session 完成 89 个连续任务、8000 万 token 的场景下验证了无可测量的任务准确率下降。这是目前看到的最系统化的"agentic 落回 KV cache/context 管理"的方案。
- **MosaicKV: Serving Long-Context LLM with Dynamic Two-D KV Cache Compression**（arXiv 2607.00760）— 现有 KV cache 压缩通常只压 sequence 维或只压 channel 维，两者一起压又会精度大跌。MosaicKV 按 KV cache segment 粒度动态选择压缩策略（利用 KV 向量内部重要性分布不均匀），并用 GPU/CPU 闲置资源管理压缩后的 cache、加速 attention 计算，缓解细粒度稀疏化带来的管理开销。是今天搜到的最扎实的系统层新工作。

### 📄 新论文

#### KV Cache 压缩/驱逐
- Forget Without Compromise: Nexus Sampling for Streaming KV-Cache Eviction Under Fixed Budgets（arXiv 2606.23961）— 指出现有 eviction 方法都是"单步 attention score + 确定性 top-K"模板，会把一次临界的低分误判变成不可逆的永久丢弃；提出 Nexus Sampling，用一种在 direct attention 上做迭代游走的打分方式找出"桥接 token"，80% 驱逐率下在 LongBench 上与 dense attention 差距 <1%，检索密集型任务上优于 top-K baseline，单序列缓存可小 10 倍。
- FreqDepthKV: Frequency-Guided Depth Sharing for Robust KV Cache Compression in Long-Context LLM Inference（arXiv 2607.06519）— 频域引导的跨层深度共享方案，用于长上下文 KV cache 压缩（未能取到完整摘要，仅记录标题/主题，供后续核对）。
- DepthWeave-KV: Token-Adaptive Cross-Layer Residual Factorization for Long-Context KV Cache Compression（arXiv 2607.06523）— 与 FreqDepthKV 同日挂出，token 自适应的跨层残差分解压缩方案，同属"跨层共享/分解"这一压缩子方向。

#### KV Cache 复用 / Position Re-indexing
- Leyline: KV Cache Directives for Agentic Inference（arXiv 2606.01065）— 指出现有 KV cache 管理假设的是"只追加"的 chatbot workload，但 agentic LLM 会因失败重试、丢弃过期输出、轨迹改道而需要策略驱动地编辑/替换已缓存的一段内容并继续执行，且不必重新 prefill 后续内容。提出声明式的 (span, replacement) 四元组把"编辑什么"和"如何保持位置正确"解耦，用架构无关接口路由到每种架构的 kernel，通过闭式 RoPE-rotation 校正恢复 attention 数学正确性——是今天除"今日必看"外，另一个直接命中"位置管理"核心问题的工作，值得和 CacheBlend/MiniPIC 对比着看。

#### Systems / Serving
- MosaicKV（arXiv 2607.00760）— 见"今日必看"。
- Towards Load-Aware Prefill Deflection for Disaggregated LLM Serving（arXiv 2607.02043）— 面向 prefill-decode 分离架构的负载感知 prefill 偏转/多路复用策略，提升高吞吐场景下的 serving 效率。
- DSpark: Confidence-Scheduled Speculative Decoding with Semi-Autoregressive Generation（arXiv 2607.05147）— 把置信度调度的推测解码和半自回归生成结合，涉及推测解码场景下的 KV cache 管理问题。
- SGLang v0.5.14（6/26）→ v0.5.15（7/10）→ v0.5.15.post1（7/14）— 相比上次记录的 v0.5.13，版本已连跳两级：v0.5.14 加入 Mamba radix cache 的 int8 checkpoint 池（线性注意力 prefix-cache 内存节省）及推测卷积窗口中间缓存去重（footprint 减半）；v0.5.15 把 Spec V2 的 CUDA-graphable DSA draft-extend 设为默认调度路径（去掉 D2H/H2D 同步，吞吐 +11%），新增 IndexShare MTP（跨 draft step 复用 indexer top-k，长上下文场景 draft-step 成本最多降 1.9×）以及融合 top-k 选择与 page-table 变换的 TopK V2。

#### Agentic Context 管理
- Beyond Compaction / CWL（arXiv 2606.11213）— 见"今日必看"。
- TokenPilot: Cache-Efficient Context Management for LLM Agents（arXiv 2606.17016）— 指出现有文本剪枝/动态内存驱逐方法会无约束地改变序列布局，造成 prefix 不匹配和 cache 失效，形成"文本稀疏性 vs prompt cache 连续性"的矛盾；提出双粒度方案：全局用 Ingestion-Aware Compaction 稳定 prompt 前缀、消除开放世界噪声，局部用 Lifecycle-Aware Eviction 按残余效用做保守的批次轮次驱逐。在 PinchBench/Claw-Eval 上隔离模式降本 61%/56%，连续模式降本 61%/87%，性能保持竞争力。
- Remember When It Matters: Proactive Memory Agent for Long-Horizon Agents（arXiv 2607.08716，Meta AI）— 提出"behavioral state decay"这个说法：长时程任务里决策相关的状态会被埋进 context 或被挤出窗口，导致该起作用时不起作用。用一个独立的 memory agent 伴随未修改的 action agent 运行，维护结构化 memory bank，决定是否注入提醒；即插即用，在 Terminal-Bench 2.0 上 pass@1 +8.3pp，在 τ²-Bench 上 +6.8pp。
- Multi-Head Recurrent Memory Agents（arXiv 2607.01523）— 讨论循环式记忆 agent 把输入迭代压缩进固定大小记忆窗口时出现的端到端性能退化问题，指出"记忆保留"是主要瓶颈，属于诊断性质的工作。
- Self-Compacting Language Model Agents（arXiv 2606.23525）— 让 agent 自身具备"自压缩"能力的架构工作，属于 tier_2 长时程 context 管理方向。
- Diagnosing and Mitigating Context Rot in Long-horizon Search（arXiv 2606.29718）— 针对 deep research / 长时程搜索场景下的 "context rot"（上下文腐化）现象做诊断和缓解，是从经验现象角度而非提出具体压缩机制的角度切入。
