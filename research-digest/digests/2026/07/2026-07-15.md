### 🔥 今日必看

- **From Tensor Buffer to Distributed Memory Hierarchy: A Survey of KV Cache Management for LLM Serving**（arXiv 2607.02574）— 本月刚挂出的综述，把 eviction/compression/offloading/reuse 统一放进"tensor buffer → 分布式内存层级"这一个框架里讲。值得拿来对照 keywords.yaml 的分层是否有遗漏，尤其是 tier_1 的 offloading 相关词。
- **Self-GC: Self-Governing Context for Long-Horizon LLM Agents**（arXiv 2607.00692）— 本月最新的 agentic context 管理工作，让长时程 agent 自己决定何时压缩/驱逐/保留 context，而不是外部固定策略触发。跟 tier_2 关注的"落回 KV cache/context 管理"高度对齐。
- **MiniPIC: Flexible Position-Independent Caching in <100LOC**（arXiv 2606.13126，watchlist 项目）— 用不到 100 行代码实现灵活的 position-independent caching，直接命中"位置墙"（多块 re-index 是否 scramble）这条核心假设，值得细读它的 position 处理方式和 CacheBlend/EPIC/MPIC 的差异。

### 📄 新论文

#### KV Cache 压缩/驱逐
- CompressKV（arXiv 2606.24467）— 语义检索引导的 GQA KV cache 压缩框架，多个内存预算下优于现有 eviction 方法。
- SeKV（arXiv 2606.31145）— 按语义 span 组织 context，用 GPU-CPU 层级避免不可逆驱逐，按需恢复 token 级细节；对"驱逐即丢失"的假设是个挑战角度。
- Value-Aware Stochastic KV Cache Eviction for Reasoning Models（arXiv 2606.03928）— 针对推理模型 decode 阶段的价值感知随机驱逐策略。
- HARD-KV（arXiv 2606.28831）— decode 阶段按 head 自适应正则化的 KV 压缩方法。
- Fail-Closed Lowering of Resident KV Claims onto LLM Serving Runtimes（arXiv 2606.01387）— 把"常驻 KV"的资源声明以 fail-closed 方式下沉到 serving runtime。

#### KV Cache 复用 / Position Re-indexing
- MiniPIC（arXiv 2606.13126）— 见"今日必看"，watchlist 项目更新。
- RedKnot（arXiv 2606.06256）— head-aware 的 KV 复用配合 SegPagedAttention，面向长上下文 serving 的复用系统。

#### Systems / Serving
- From Tensor Buffer to Distributed Memory Hierarchy 综述（arXiv 2607.02574）— 见"今日必看"。
- vLLM v0.25.0 / v0.25.1（7/11、7/14 发布）— prefix caching 支持 Mamba hybrid 模型，KV offloading tiering metric plumbing，批量 lookup 下沉到 C；上一版 v0.24.0 加了 KV-cache watermark 减少 preemption、跨组 prefix-cache 命中的两阶段分配。
- LMCache v0.5.1（7/6 stable）+ nightly（7/14）— multi-process 协调器新增 L2→L1 warm-prefetch，新增 HiddenStateStore 缓存 hidden states，AMD hipFile GDS L1 后端、MUSA 多进程 handle wrapper、XPU docker 镜像。
- SGLang v0.5.13 — Spec V2（tree drafting, topk>1）在 triton/FA3/MLA/aiter 后端全部转正为默认路径；HiCache 对 hybrid 模型（SWA/Mamba）默认开启；TokenSpeed MLA 后端在 Blackwell 上支持 FP8 KV cache。

#### Agentic Context 管理
- Self-GC（arXiv 2607.00692）— 见"今日必看"。
- LLM Agents Are Latent Context Managers: Eliciting Self-Managed Context via a Proprioceptive Dashboard（arXiv 2606.30005）— 给 agent 一个"本体感受仪表盘"，让它自己感知并管理自身 context 状态。
- ACE: Pluggable Adaptive Context Elasticizer across Agents（arXiv 2606.31564）— 可插拔的跨 agent 自适应 context 伸缩机制。
- IntentKV（arXiv 2606.09916，watchlist 项目）— 冻结 base LLM，用跨轮次 QueryMemory 给历史 token 打分做驱逐，8k KV 预算下 token 丢弃率降 23.9%-30.7%（Qwen3-8B）。
- SideQuest（arXiv 2602.22603，watchlist 项目）— 让 LRM 自己推理判断 context 里哪些 token 有用来做 KV cache 压缩，面向多轮 tool-calling + 内部反思的 agentic 场景。
